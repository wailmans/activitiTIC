Les techniques de l’information et de la communication (TIC):


TIC signifie utiliser des outils pour partager et modifier l'information. Les ordinateurs, les téléphones, Internet et la télévision font
tous partie des TIC. Nous pouvons les utiliser pour parler avec des personnes éloignées, trouver des informations en ligne ou faire des
choses comme des photos ou des vidéos.


Catégories et objectifs:


TIC est divisé en trois catégories:la téléinformatique (qui comprend des choses comme Internet et les serveurs), les télécommunications (comme les téléphones portables et les systèmes GPS) et l'électronique. Il y a cinq objectifs que les gens veulent atteindre avec ces technologies : connectivité, conception modulaire, installation simple, utilisation facile pour tout le monde sans avoir besoin de connaissances particulières des réseaux ou des structures logicielles, ainsi que trouver des erreurs dans la transmission des informations afin qu'elles puissent être corrigées rapidement.


Composants de TIC:


1. La partie matérielle


  • Pc portable, la télévision, le téléphone portable, appareils photos, les robots...
  • Station de travail : toute machine capable d’envoyer des données vers les réseaux (PC, MAC, SUN Terminal X, ...),

2. Caractéristiques

  • La rapidité : dans la réalisation des activités.
  • La décentralisation du pouvoir: elles permettraient les agents à recevoir les informations se référant à son domaine
  • Un meilleur accès aux informations
  • La rationalité : donner un avantage pour l'entreprise
  • La mondialisation
  • La répartition : l’information circule entre un nombre très élevé de stations avec une qualité excellente L’intensité

Initiation aux techniques de communication


Les objectifs de communication :


Les objectifs de communication sont les réactions attendues du public cible. Ils doivent être définis selon trois dimensions :
cognitives, affectives et comportementales.


Un système de communication comprend les éléments suivants :


  • Les partenaires de communication : l'émetteur
  • Les vecteurs : le message et les médias.
  • Les fonctions : le codage, le décodage, la réponse et le feedback.
  • Le bruit : les perturbations pouvant altérer les signaux pendant la transmission.

Il existe deux types de communication :


  1. La communication institutionnelle (ou corporate) qui vise à promouvoir l'image de marque de l'entreprise ou de l'institution.
  2. La communication commerciale ou de produit qui met en avant les caractéristiques.

Il existe trois types de schémas de communication :



  1. La communication interpersonnelle : elle est utilisée lorsqu'un émetteur communique avec un destinataire.
  2. La communication de groupe : elle est utilisée lorsqu'un émetteur communique avec plusieurs destinataires.
  3. La communication de masse.

Média et Multimédia :


  1. Un média : est un moyen impersonnel de diffusion d'informations à grande échelle.
  2. Le multimédia : fait référence à l'utilisation simultanée et interactive de plusieurs modes de représentation de l'information.

Réseau informatique :


Réseau (informatique):


Un réseau informatique est un ensemble d'ordinateurs et de terminaux connectés entre eux pour échanger des informations numériques. L'objectif d'un réseau est de permettre la communication de données entre applications, quelles que soient les distances qui
les séparent. Les réseaux ont évolué pour inclure non seulement des données, mais également la parole et la vidéo.


Les caractéristiques de base d'un réseau:


  • Topologie : la façon dont les équipements sont interconnectés (physique) et communiquent entre eux (logique).
  • Débit : quantité de données numériques transmises par seconde.
  • Distance maximale : portée du réseau, dépendant de la technologie utilisée.
  • Nombre maximum de nœuds : nombre maximum d'équipements que l'on peut interconnecter.

Quelques terminologies en réseau


  • Station de travail : une machine capable d'envoyer des données.
  • Nœud : une station de travail, une imprimante, un serveur ou une entité adressable.
  • Paquet : la plus petite unité d'information sur le réseau contenant l'adresse de l'émetteur, du récepteur et les données à transmettre.
  • Réseaux homogènes : tous les ordinateurs sont de même constructeurs.
  • Réseaux hétérogènes : les ordinateurs sont de constructeurs divers.
  • Débit : la quantité de données transmises par seconde (bit/s ou bps).

Éléments d'un réseau


  • Ordinateurs avec une carte de communication
  • Logiciels tels que navigateurs web, clients de messagerie, serveurs web, etc.
  • Supports de communication tels que câbles de paires torsadées, prises RJ45, WiFi, etc. pour le LAN, et ligne téléphonique,
    ADSL, fibre optique pour le WAN
  • Équipements d'interconnexion tels que répéteurs, concentrateurs, commutateurs pour le LAN, et routeurs pour le WAN.

Classification des réseaux


Il existe plusieurs façons de classer les réseaux informatiques. Les deux principales classifications sont basées sur leur étendue
et leur topologie.


Classification basée sur l'étendue:


    Selon leur étendue, on peut classer les réseaux en :

  • réseau local (LAN - Local Area Network)
  • réseau étendu (WAN - Wide Area Network)
  • réseau métropolitain (MAN - Metropolitan Area Network)

Classification basée sur la topologie


Selon leur topologie, on peut classer les réseaux en :


  • réseau en bus (Bus)
  • réseau en anneau (Ring)
  • réseau en étoile (Star)
  • réseau maillé (Mesh)

Il existe également d'autres classifications basées sur des critères différents, tels que le mode de transmission, le type de support, le protocole de communication, etc.

Les Composants matériel d’un Réseau informatique


  • La carte réseau : est un composant matériel qui permet à un ordinateur de se connecter à un réseau en utilisant des protocoles
    de communication spécifiques
  • Le switch est un composant matériel qui permet à plusieurs appareils de communiquer entre eux en utilisant des adresses MAC et offre
    des fonctionnalités avancées telles que la gestion des VLAN, la qualité de service (QoS), etc
  • Le hub est un composant matériel qui permet de connecter plusieurs appareils sur un réseau local, fonctionne comme un répéteur et
    transmet les données à tous les autres appareils connectés. Cependant, il ne possède pas la fonctionnalité de la table de correspondance
    et est largement remplacé par des switches

Le Modèle OSI


  • Le modèle OSI : est un modèle de référence pour les communications de réseau, divisant les communications en sept couches distinctes.
    Chaque couche est responsable de fonctions spécifiques liées à la communication de données entre les appareils de réseau

    1. la couche liaison de données
    2. la couche réseau
    3. la couche transport
    4. la couche session
    5. la couche présentation
    6. la couche application

  • Bien qu'il ne soit pas utilisé dans les protocoles de communication de réseau modernes tels que TCP/IP, le modèle OSI reste utile pour
    comprendre les principes fondamentaux de la communication de réseau.

Le Modèle TCP/IP


  • Le modèle OSI et le modèle TCP/IP sont deux modèles de référence pour les communications de réseau.
  • Le modèle OSI divise les communications en sept couches, tandis que le modèle TCP/IP divise les communications en quatre couches
    principales.
  • Le modèle TCP/IP est utilisé pour la conception et la mise en œuvre des réseaux modernes, tels que les réseaux d'entreprise,
    les réseaux Internet et les réseaux sans fil.

L’adressage


L'adressage dans un réseau permet d'identifier de manière unique les unités en communication.


Une adresse IP (Internet Protocol) est utilisée pour identifier chaque ordinateur dans un réseau.

Il existe deux modes d'adressage :


  • IPV4 qui utilise des adresses de 32 bits, et IPV6 qui utilise des adresses de 128 bits.
  • Les adresses IPV4 et IPV6 permettent d'identifier de manière unique les ordinateurs dans un réseau.

L’adresse IPV4


Les adresses IPV4 sont composées de deux parties : l'adresse réseau et l'adresse hôte. La partie réseau est identique pour
tous les hôtes d'un réseau, tandis que la partie hôte varie. Le nombre d'hôtes possibles est déterminé par le nombre de bits dans
la partie hôte.

Les types d'adresses IPV4 sont :

  • Adresse réseau (bits de partie hôtes = 0),
  • Adresse de diffusion (bits de partie hôtes = 1),
  • Adresse hôte (adresses entre l'adresse réseau et l'adresse de diffusion),
  • Masque de réseau (bits de partie réseau = 1).

Introduction:


Git est un logiciel de versioning créé en 2005 par Linus Torvalds, le créateur de Linux. Il permet de conserver un historique des modifications effectuées sur un projet
afin de pouvoir facilement identifier les changements effectués et de revenir à une ancienne version en cas de problème. Les logiciels de gestion de versions sont
quasiment incontournables aujourd'hui car ils facilitent la gestion de projets et permettent de travailler en équipe plus efficacement.


A quoi sert concrètement un système de gestion de version?


L’organisation du travail est beaucoup plus complexe pour assurer que les fonctionnalités des développeurs ne sont pas en conflit avec les autres, que chaque
développeurpossède une version actualisée de la site pour tester et implémenter ses fonctionnalités. Les logiciels de versioning et Git offrent un système de gestion de version décentralisé rudimentaire.


Les deux modèles des logiciels de gestion de version :
  modèle centralisé vs modèle décentralisé.


Git permet de coordonner le travail entre plusieurs personnes en conservant un historique des changements effectués sur des fichiers.
Les développeurs travaillant avec Git ont accès à l'historique des modifications pour l'intégralité du projet.


Qu'est-ce que GitHub?


Ce dernier est un programme de gestion de version qui est mixé d'un modèle centralisé et un modèle décentralised. Le principe de base d'un modèle centralisé est
la centralisation du code source lié au projet, whereas le modèle distribué laisse l'intégralité du code source sur un serveur distant. Les avantages principales de
l'exemple centralisé sont son simplicity/flexibilité du travail et la sécurité de l'utilisateur, qui permet d'utiliser la copie du projet d'un utilisateur comme backup
en cas de corruption du serveur centrale.


Utiliser Git : ligne de commande, console et interface graphique.

La gestion des informations selon Git:


Git gère les données en utilisant une approche de flux d'instantanés ou "snapshots". À chaque enregistrement de l'état d'un projet, Git prend un instantané du
contenu de l'espace de travail et l'enregistre localement dans une base de données. L'historique complet du projet étant disponible localement, la plupart des
opérations de Git peuvent être effectuées localement, ce qui les rend rapides et efficaces. En somme, Git est un système de gestion de version performant offrant
de nombreuxavantages pour la gestion des informations dans un projet.


Les états des fichiers:


Git suit les modifications des fichiers dans un projet en utilisant trois états : "modifié", "indexé" et "validé". Au démarrage d'un dépôt Git, tous les fichiers
sont non suivis et doivent être indexés et validés. Lors du clonage d'un dépôt existant, les fichiers sont déjà suivis et validés par défaut. Un fichier est
modifié tant qu'il n'a pas été indexé, et un nouveau fichier est non suivi jusqu'à ce qu'il soit indexé. L'indexation indique à Git que le fichier modifié ou nouveau
doit être inclus dans le prochain instantané. Lorsqu'on demande à Git de prendre un instantané, les fichiers indexés et non modifiés sont validés
et le cycle recommence.

  • le répertoire Git :qui stocke les informations concernant ce qui fera partie du prochain instantané ou du prochain "commit

Qu’est ce qu’une branche ?


Git a une approche différente des branches qui rend la création de nouvelles branches et la fusion de branche très facile à réaliser. Une branche est une simple fichier contenant
les 40 caractères de l'empreinte SHA-1 du commit sur lequel elle pointe. Lorsqu'on effectue un commit, Git stocke un objet commit qui contient les nom et prénom de l'auteur du commit, le message renseigné lors de la création du commit ainsi que un pointeur vers l'instantané du contenu indexé et des pointeurs vers le ou les commits précédant directement le commit courant. Un pointeur est un objet qui contient l'adresse d'une donnée stockée quelque part. La branche master va se déplacer automatiquement à chaque nouveau commit pour pointer sur le dernier commit effectué tant que on reste sur cette branche.
